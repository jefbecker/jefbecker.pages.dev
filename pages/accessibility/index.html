Accessibility

Every now and then, accessibility is a neglected topic on the internet. When I started building this site, accessibility didn't even cross my mind, but I soon realized its importance and that I could make something more accessible even if it cost me a lot more learning time.

As useless as it may be, I want this space to be as accessible as possible. Fortunately, I believe I have achieved my goal.

Accessibility Features

My main goal is to achieve at least Level AA of the Web Content Accessibility Guidelines (WCAG) 2.0. The following features are available:

  • Proper HTML language semantics;
  • Full keyboard navigation;
  • Image description in non-decorative elements;
  • Automatic light and dark modes according to user preferences;
  • The site uses the default font of your operating system;
  • The layout adjusts according to zoom level without issues;
  • Links are described in most cases;
  • The site is responsive and should work on any screen size;
  • No navigation feature requires JavaScript to work;
  • Navigation compatible with screen readers;
  • The color palette follows a minimum contrast ratio of 4.5:1;
  • The site can be read in two languages: English and Brazilian Portuguese.

All these features have been tested by me on different devices and automated verification tools:

  • Web Accessibility Checker;
  • WAVE Web Accessibility Evaluation Tools;
  • Color Contrast Accessibility Validator;
  • Screen reading was tested on the Edge browser reader, it seems to work well.

I cannot fail to mention those who inspired me to do all this:

  • Frills;
  • Nonkiru;
  • Moxie.

This site is entirely amateur, I may/will make mistakes in some aspect. Please get in touch if you find something broken or not in accordance with the practices mentioned above. And if you use any accessibility tool, please contact me as well! I would like to know how I did and what I can do to improve.
